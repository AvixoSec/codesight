CodeSight Docs

CodeSight scans code, verifies scanner alerts, and writes reports that CI can use.

01

Start

Install, run the guided terminal UI, and scan a file without memorizing flags.
02

Verify SARIF

Import Semgrep, CodeQL, or other SARIF alerts and attach local source evidence.
03

CI

Write SARIF, choose a fail threshold, and upload results to code scanning.
04

Benchmarks

Use the current dataset as a smoke test, not as a public comparison claim.

What Changed

CodeSight is no longer only "scan this file and tell me what is wrong." The main flow now starts with scanner alerts, pulls in source context, and asks for evidence before raising confidence.

Guided UI

Run codesight and choose scan, diff, verify, artifact bundle, judge, or skeptic mode.

SARIF Verify

Import scanner alerts, resolve paths safely, and keep plain imports as uncertain.

Judge Mode

Use your configured provider to check source, sink, missing guard, evidence path, and fix.

Skeptic Pass

Review serious verdicts again so weak evidence can be downgraded before CI blocks.

Proof Bundle

Write Markdown, JSON, SARIF, and a manifest into one artifact folder.

Direct Scan

Still scan files, folders, and git diffs when you do not have scanner SARIF yet.

First Commands

Terminal
$ pip install codesight
$ codesight
$ codesight security src/app.py
$ semgrep scan --config auto --sarif > semgrep.sarif
$ codesight verify semgrep.sarif --source . --output markdown

Verdicts

  • exploitable - enough evidence to block
  • likely_exploitable - strong evidence, small uncertainty
  • uncertain - needs review
  • probably_false_positive - probably safe, with a reason
  • not_exploitable - dismissed with evidence
