Architecture

Small modules, plain data, and conservative verdicts.

Main Flow

  1. CLI parses the command or opens the guided UI.
  2. Scanner SARIF or source files are loaded.
  3. Context is resolved from the local source root.
  4. Findings are normalized into CodeSight verdicts.
  5. Optional judge and skeptic passes update evidence.
  6. Formatters write Markdown, JSON, plain text, or SARIF.
  7. Artifacts can be written as a proof bundle.

Modules

ModuleJob
cli.pyCommands, guided UI, and orchestration.
sarif.pySARIF import and export.

findings.pyStructured verdicts, evidence, and finding models.
judge.pySemantic judge and skeptic prompts.
profiles.pyFramework hints for Flask, FastAPI, Express, Django, actions, and agents.
formatters.pyMarkdown, JSON, and plain report output.
artifacts.pyProof bundle writer.
providers/OpenAI, Anthropic, Google, Ollama, and compatible APIs.

Safety Choices

  • Plain SARIF import stays uncertain.
  • Paths cannot escape the chosen source root.
  • Missing files lower confidence instead of being ignored.
  • Project config cannot override keys, base URLs, or default provider.
  • CI threshold defaults to confirmed exploitable findings.

Provider Boundary

Provider calls only happen when a command needs them. Preview mode and plain import mode can run without sending source to a hosted provider.
