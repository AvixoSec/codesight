Security Policy

CodeSight is a defensive security tool. Here's how we handle security for the tool itself.

Reporting Vulnerabilities

If you find a security vulnerability in CodeSight, do not open a public issue.

Email AvixoSec@gmail.com with:

  • Description of the vulnerability
  • Steps to reproduce
  • Impact assessment
  • Suggested fix (if any)

We'll respond within 48 hours and coordinate disclosure.

Scope

CodeSight is strictly a defensive cybersecurity tool. It:

  • Does not execute untrusted code
  • Does not exfiltrate data
  • Has no offensive capabilities

Data Handling

  • API keys are stored locally in ~/.codesight/config.json
  • Source code is sent only to the configured LLM provider API
  • No telemetry, no analytics, no phone-home
  • Ollama mode keeps everything on your machine — zero network calls
