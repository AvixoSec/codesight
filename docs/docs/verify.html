Verify SARIF

Import scanner alerts, add source context, and turn noise into verdicts.

Run It

Terminal
$ semgrep scan --config auto --sarif > semgrep.sarif
$ codesight verify semgrep.sarif --source . --output markdown

Modes

ModeUse it for
--preview-contextSee the exact source snippets before a provider sees them.
--judgeAsk the configured provider to decide from evidence.
--skepticRecheck serious judge findings and downgrade weak evidence.
--artifact-dirWrite report.md, report.json, results.sarif, and manifest.json.

Verdicts

  • exploitable - enough evidence to block
  • likely_exploitable - strong evidence, small uncertainty
  • uncertain - imported or partially checked, needs review
  • probably_false_positive - probably safe, with a reason
  • not_exploitable - dismissed with evidence

Fail Threshold

Default threshold is exploitable. Plain import mode does not fail CI on uncertain.

Terminal
$ codesight verify semgrep.sarif --source . --fail-on exploitable
$ codesight verify semgrep.sarif --source . --fail-on likely_exploitable
$ codesight verify semgrep.sarif --source . --fail-on never

Profiles

Profiles give the judge framework hints. They do not replace source evidence.

auto, generic, flask, fastapi, express, django, github-actions, and ai-agent are supported.

Local Fixture

Terminal
$ codesight verify examples/semgrep-verify/semgrep.sarif \
  --source examples/semgrep-verify/project \
  --profile flask \
  --output markdown
