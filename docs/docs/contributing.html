Contributing

Keep changes focused, tested, and easy to review.

Setup

Terminal
$ git clone https://github.com/your-org/codesight.git
$ cd codesight
$ python -m venv .venv
$ pip install -e ".[dev]"

Checks

Terminal
$ pytest tests -v
$ python -m ruff check codesight tests
$ python -m mypy codesight/ --ignore-missing-imports

Good Pull Requests

  • Explain what changed.
  • Explain why it matters.
  • Include the checks you ran.
  • Add fixtures when verify, SARIF, judge, or formatters change.
  • Keep unrelated cleanup out of the same pull request.

Useful Work

  • More SARIF fixtures.
  • Better framework profiles.
  • Cleaner proof bundles.
  • Benchmark cases with expected evidence paths.
  • Provider health checks and error messages.
