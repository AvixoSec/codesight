CI/CD

Use CodeSight as a SARIF verifier first. Direct scans are still available.

Recommended Flow

Let a fast scanner find breadth. Let CodeSight check the alert against source context and decide whether CI should block.

GitHub Actions
- run: |
    python -m pip install semgrep
    semgrep scan --config auto --sarif --output semgrep.sarif

- uses: your-org/codesight@v0.3.0
  with:
    mode: verify
    path: .
    sarif-input: semgrep.sarif
    output: sarif
    fail-on: exploitable
    judge: "true"
    skeptic: "true"
    profile: auto
    provider: openai
    api-key: ${{ secrets.OPENAI_API_KEY }}

Direct Scan

GitHub Actions
- uses: your-org/codesight@v0.3.0
  with:
    provider: openai
    api-key: ${{ secrets.OPENAI_API_KEY }}
    task: security
    path: .
    output: sarif

Thresholds

ValueBehavior
exploitableFail only on confirmed exploitable findings.
likely_exploitableFail on confirmed and likely exploitable findings.
uncertainAlso fail on alerts that still need review.
neverAlways exit zero after writing reports.

Artifact Bundle

For audits or grant demos, write all outputs into one folder.

Terminal
$ codesight verify semgrep.sarif --source . --artifact-dir .codesight-proof

Pre-commit

.pre-commit-config.yaml
repos:
  - repo: https://github.com/your-org/codesight
    rev: v0.3.0
    hooks:
      - id: codesight-security
