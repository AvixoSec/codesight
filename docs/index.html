Линтер видит синтаксис. CodeSight видит смысл - и те баги, что доезжают до прода.

Утилита командной строки. Прогоняет твой код через большие языковые модели и вытаскивает реальные дефекты: SQL-инъекции, захардкоженные секреты, обход аутентификации, гонки потоков. Отдаёт SARIF для CI, CWE ID для аудиторов и понятный текст - для людей.

01 · Введение

В чём идея

Статический анализатор скажет, что не хватает закрывающей скобки. Он не скажет, что пользовательская строка долетает до execute() без параметризации, что секрет закоммичен в открытом виде, что два потока дерутся за один и тот же файловый дескриптор. Это семантика - и именно такие баги последние тридцать лет ломают прод.

В отличие от лексики и синтаксиса - это баги, которых парсер не видит.

CodeSight - небольшая CLI, которая отдаёт семантическую часть работы LLM, а результат заворачивает в форматы, которые уже понимают твои пайплайны: SARIF, JSON, Markdown. И уходит в сторону. У неё есть мнение про что спрашивать у модели - но не про то, какую модель использовать. Ключ твой.

SARIF - OASIS-стандарт. Парсится напрямую в GitHub Advanced Security и SonarQube.
“Линтер видит синтаксис. CodeSight видит смысл.” - однострочное описание проекта, не менялось с версии 0.1

02 · Команды

Девять команд. Один бинарник.

Каждая команда берёт файл, директорию или поток с пайпа. Формат вывода переключается флагом, дефолты разумные. Никаких подкоманд от подкоманд.

Вся CLI спроектирована так, чтобы помещаться в --help. Если не держится в голове - слишком сложно.
01

review

Ревью уровня сеньора: архитектура, читаемость, скрытые допущения.

codesight review
02

bugs

Логические дефекты, которые прошли мимо тестов - off-by-one, null-разыменования, утечки ресурсов.

codesight bugs api.py
03

security

Аудит уязвимостей с привязкой к CWE и OWASP Top-10. На выходе SARIF.

codesight security src/
04

docs

Докстринги, саммари модулей, куски README прямо из исходников.

codesight docs core/
05

explain

Пересказ легаси на нормальном языке. Меньше археологии.

codesight explain legacy.c
06

refactor

Конкретные переписи в виде unified diff - не размытые советы.

codesight refactor
07

scan

Рекурсивный обход репозитория. Поддерживает смарт-контракты на Solidity.

codesight scan .
08

diff

Ревью только изменений в git-дельте - идеально для CI.

codesight diff HEAD~1
09

benchmark

Сравнить находки разных провайдеров на одном промпте.

codesight benchmark

03 · Сравнение

Рядом с классикой

CodeSight не заменяет regex-линтер или точный анализатор вроде CodeQL - он их дополняет. Классика хороша на синтаксических правилах и taint-анализе. CodeSight читает код так, как читает сеньор: с контекстом, намерением и сомнением.

Что умеет Линтер (semgrep) CodeQL CodeSight
Проверка синтаксиса Да Да Через LLM
Захардкоженные секреты По regex Да Семантика + контекст
Обход аутентификации Нет Частично Да
Состояния гонки Нет Частично Да
SQL-инъекции через f-string Частично Да Да
Баги бизнес-логики Нет Нет Да
Объяснения словами Нет Нет Да
Свой провайдер Нет Нет 16+ моделей
Цена запуска Часы Дни 60 секунд

Не замена классическому SAST - дополнение к нему.

04 · Провайдеры

Свой ключ, своя модель

CLI не привязана к провайдеру. В коробке четыре штатных интеграции, плюс работает с любым эндпоинтом, совместимым с OpenAI - через общий адаптер.

API-ключ остаётся на твоей машине. У CodeSight нет своего сервера, чтобы куда-то что-то отправлять.
OpenAI
gpt-5.4
Дефолт. Быстрее всех по wall-clock и самая точная с одного прохода на наших бенчмарках.
Anthropic
claude-opus-4.6
Глубже всех рассуждает. Лучший на тонких логических ошибках, где нужно несколько шагов вывода.
Google
gemini-3.1-pro
Самое большое контекстное окно. 50 тысяч строк в один запрос - без саммари.
Ollama
llama3 · codellama · mistral
Локально. Ни egress, ни API-ключей, ни оплаты за файл. Аудит можно делать из air-gapped среды.

Ещё совместимо - любой OpenAI-совместимый эндпоинт:

OpenRouterGroqAzure OpenAIDeepSeek xAITogetherMistralFireworks PerplexityCerebrasCohere

05 · Архитектура

Как это работает - на одной странице

Маленький пайплайн, специально. CodeSight читает файл, собирает промпт с нужным контекстом, вызывает сконфигуренный провайдер и парсит структурированный ответ в объекты-находки, не зависящие от языка.

Цепочка triage + verify сокращает стоимость на больших репозиториях примерно на 60%: уверенные вердикты проходят без дорогой верификации.

08 · До / после

Те же четыре строки, другое прочтение

Линтер видит валидный Python. Тайп-чекер видит совместимые типы. Тесты зелёные. Код уходит в релиз. Через месяц пользователь обнаруживает, что может вытащить всю таблицу users, если подсунуть хитрый uid. CodeSight читает те же четыре строки и не пропускает их на ревью.

До · статический анализ
Вывод линтера по user_handler.py:
# All checks passed. 0 issues.
@app.route("/user/<uid>")
def user(uid):
    return db.query(f"SELECT * FROM u WHERE id={uid}")
После · codesight security
Находки CodeSight по тому же файлу:
@app.route("/user/<uid>")
def user(uid):
    return db.query(f"SELECT * FROM u WHERE id={uid}")
    ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    CRIT · CWE-89 · SQL injection via f-string
    Фикс: используй параметризованный запрос

Это сокращённый пример. В реальном выводе будет полный фрагмент с фиксом, маппинг на OWASP и уровень уверенности верифицирующей модели.

09 · Решения

Четыре тихих решения

CLI, а не обёртка над SaaS

Никакой панели, никакого сервиса. UI - твой шелл. Выхлоп - текст.

Нейтральность к провайдеру

Ключ твой. Модель переключается флагом под каждую команду.

SARIF по умолчанию

У каждой находки есть CWE, OWASP и severity - пайплайн это уже понимает.

Офлайн-режим

Направь на локальный Ollama. Никакого egress, никаких логов, никаких сюрпризов.

11 · Вопросы

Что спрашивают чаще всего

Пять коротких ответов на то, что регулярно прилетает в GitHub issues. Если твоего вопроса тут нет - заведи discussion.

Чем отличается от Copilot?

Copilot пишет код за тебя. CodeSight читает уже написанный и говорит, что с ним не так. Другой цикл, другая задача.

Copilot - это автокомплит с мнением. CodeSight - ревьюер с маппингом на CWE. Они нормально уживаются, можно использовать оба на одном файле.

Вы логируете мой код?

Нет. У CodeSight нет сервера. Код уходит только туда, куда ты сам настроил провайдера. API-ключ видим только локально, никаких обращений на наш бэк не делается - потому что его нет.

Если не хочешь вообще никакого egress - возьми провайдер ollama и покажи на локальную модель. Ничего не уходит за границу машины.

Можно использовать свои промпты?

Да. У каждой встроенной команды промпт лежит в читаемом Jinja-шаблоне в codesight/templates/. Скопируй, поправь, подсунь через --template path/to/custom.j2.

Подкоманда codesight prompts для управления именованными вариантами - в issue #47.

Почему не CodeQL?

Бери оба. CodeQL даёт точные результаты на основе правил и почти без ложных срабатываний. CodeSight даёт гибкий анализ с объяснениями и контекстом. Эти два набора находок редко полностью совпадают.

CodeQL силён на известных классах уязвимостей, где можно описать taint-путь. CodeSight силён на бизнес-логике и идиомах, под которые ещё никто не написал запрос.

А rate limits?

CodeSight уважает заголовок Retry-After и откатывается экспоненциально. Для больших сканов можно задать лимит конкурентности и бюджет токенов в минуту в ~/.codesight/config.toml.

Нужен выше throughput - заведи Groq или Cerebras, оба работают через OpenAI-совместимый адаптер.
